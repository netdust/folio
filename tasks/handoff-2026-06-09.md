### 2. Kanban drag — SIX bugs found by real-browser smoke + fixed (post-merge)
The drag feature shipped behind a green suite but didn't work end-to-end (jsdom can't drive
real dnd-kit, so none surfaced in unit tests). Each root-caused with live DOM evidence:
- `a8d820b` — card rendered BEHIND columns → portaled `<DragOverlay>`; within-column reorder
  didn't register → `collisionDetection={closestCorners}`.
- `d869b1b` — reorder only worked in Manual sort → **auto-switch to Manual on a reorder-intent
  drag** (`auto-manual-reorder` action; persists `sort:[]`).
- `32ee429` — drop snap-back → suppress dragged-card layout animation + optimistic re-sort by
  board_position (`sortByBoardPosition` in `documents.ts onMutate`).
- `9c18c9e` — a just-moved card couldn't be reordered until you dragged another → stale dnd-kit
  measurement → `measuring={{ droppable: { strategy: MeasuringStrategy.Always } }}`.
  (Reverted a WRONG earlier `pendingSlugs` drag-block — it treated a symptom.)
- `a14f4ab` — down-reorder flicker (duplicate card) → `dropAnimation={null}` (the 180ms overlay
  fade was the duplicate window; optimistic re-sort already places the card on the release frame).
- `6693d4d` — down-by-ONE no-op → **direction-aware drop slot** in `dropSlotPosition`: moving
  down drops AFTER the over-card (`overIdx+1`), up keeps drop-before. (Was pure drop-before →
  down-by-1 landed in the card's own slot.)

## Architecture clarified this session (so it's not re-litigated)

Three tiers, deliberately separate:
- **`/link`** — pure client-side document search, no AI.
- **`/draft` `/summarize` `/decompose`** — a DIRECT one-shot AI API call (`/ai/complete`,
  `tools:[]`, read-only, returns text the editor inserts). NO operator, NO agent, NO skills, NO
  tools. Uses the single production model (`operator_model` setting). This is the safe
  can't-misbehave tier — keep it that way.
- **The operator (cockpit chat)** — a real agent run through the runner, with the 16 folio tools
  + skills + the irreversible-op confirm gate. THIS is where skills/tools/research live.
