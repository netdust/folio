# Phase 1.5 вҖ” UX Polish Design

**Date:** 2026-05-12
**Branch target:** `phase-1.5/ux-polish` (off `phase-1/frontend` HEAD `3dd3346`)
**Predecessor:** Phase 1 frontend (Tasks 5вҖ“30 shipped 2026-05-12)

## 1. Why this spec exists

Phase 1 frontend shipped 25 functional tasks but the rendered UI does not match the canonical mockups in `.superpowers/brainstorm/53908-1778511064/` and `.superpowers/brainstorm/94899-1778514720/`. Components consumed the design tokens minimally вҖ” text-only labels in the rail, no Lucide icons anywhere, status displayed as solid pills without dots, "LoadingвҖҰ" plain text instead of skeleton placeholders, empty states without iconography or rich CTAs, no `+ New work item` primary affordance (Cmd-K-only creation), `<button type="button">` outside a `<form>` on /login (no Enter-to-submit), no signup UI.

Manual QA against the running app caught three latent bugs the test suite missed (`useProject` URL drift, `<main className="max-w-5xl">` clamping the Shell, login Enter-doesn't-submit). The Phase 1 acceptance block in `docs/PHASES.md` was deliberately left unchecked pending this work.

This spec defines a polish phase that brings every Phase 1 surface up to mockup fidelity, bundles the three foundational fixes, and sets the explicit acceptance gate that lets `phase-1: complete` get committed.

## 2. Locked decisions

These decisions were made during brainstorming. Do not re-litigate.

- **Mockup fidelity is the visual target.** All 8 substantive HTMLs in the two `.superpowers/brainstorm/...` directories. Where mockups conflict, `final-design-language.html` and `folio-coreoss-shell.html` win.
- **Approach A вҖ” primitives first, then per-surface.** Two waves of subagent dispatching. Wave 1 ships isolated, testable primitives. Wave 2 composes them into surfaces.
- **Iconography:** `lucide-react` (already installed). Custom `<Icon>` wrapper enforces stroke-width 1.5 and currentColor. No raw `<svg>` in feature code; no direct `lucide-react` imports outside the wrapper.
- **Theme default:** follow `prefers-color-scheme` (already wired in Phase 0.5). User can flip via Cmd-K вҶ’ Toggle theme.
- **Rail:** support both expanded (200px) and collapsed (64px) variants. Collapsed toggle already exists via `useRailCollapsed()`. Use icon set per `rail-icons-v2.html`.
- **Signup:** new "Sign up" tab on `/login`. Uses existing `useRegister` hook (already exported, never wired).
- **Create affordances:** `+ New work item` and `+ New page` live in `MainFrame.actions` slot for each project view. Empty states show the same primary CTA. Cmd-K stays as the keyboard-fast path.
- **Empty states:** Lucide icon (32px) + heading + body + primary CTA. Five surfaces total (no workspaces, no projects, no work items, no pages, no kanban cards in any column).
- **Loading states:** Skeleton placeholders that mirror the loaded shape. Text-only `LoadingвҖҰ` is removed everywhere.
- **Field-name validation in `+ Add field`:** accept raw strings; YAML round-trip handles quoting. No slugify enforcement.
- **Optimistic create:** none. The new doc shows up in the list cache after the POST returns. Sub-100ms locally; reassess on a deployed instance later.
- **Acceptance gate:** visual sign-off (11 screenshots) + 15-scenario manual QA pass + Vitest suite + clean build.

## 3. Architecture

### 3.1 Two-wave dispatch

```
Wave 0 (3 fix commits)  вҶ’  Wave 1 (6 primitive commits)  вҶ’  Wave 2 (6 surface commits)  вҶ’  Closeout
```

Wave 0 lands the foundational fixes that block visible behavior. Wave 1 ships isolated, testable primitives. Wave 2 consumes them across the 6 visible surfaces. Closeout is the screenshot pass + 15-scenario manual QA pass + the original Phase 1 acceptance ticks.

## 4. Wave 0 вҖ” Bundled fixes

These three fixes have already been edited in the working tree but not committed; they ship first as discrete `fix:` commits before any polish work begins.

### 4.1 `useProject` path normalization

`apps/web/src/lib/api/projects.ts:34` had `client.get<Project>(\`/api/v1/w/${wslug}/projects/${pslug}\`)`. ## 5. Wave 1 вҖ” Primitives

### 5.1 `<Icon>` (new)

`apps/web/src/components/ui/icon.tsx`. Thin wrapper around any `lucide-react` icon component. Enforces `strokeWidth={1.5}`, sizes via `size?: 14 | 16 | 20 | 24 = 16`, accessibility via mutually-exclusive `label` (sets `aria-label`) or absence (sets `aria-hidden=true`). Always passes `className="shrink-0"` plus user `className`.

```tsx
import { type LucideIcon } from 'lucide-react';
import { cn } from './cn.ts';

interface Props {
  icon: LucideIcon;
  size?: 14 | 16 | 20 | 24;
  className?: string;
  label?: string;
}

export function Icon({ icon: I, size = 16, className, label }: Props) {
  return (
    <I
      strokeWidth={1.5}
      width={size}
      height={size}
      aria-label={label}
      aria-hidden={label ? undefined : true}
      className={cn('shrink-0', className)}
    />
  );
}
```

Tests (3): renders the passed Lucide icon SVG, sets stroke-width to 1.5, aria-label vs aria-hidden mutual exclusion.

### 5.2 `<Pill>` (upgrade вҖ” variant prop)

`apps/web/src/components/ui/pill.tsx`. Add `variant?: 'dot' | 'solid' = 'solid'`. Solid keeps the existing pale-background pill (used in selectors, frontmatter cells, no behavior change for current 11 sites). Dot mode renders a colored dot + text label, used by list rows and kanban column headers.

```tsx
type Category = 'backlog' | 'unstarted' | 'started' | 'completed' | 'cancelled';

interface PillProps {
  category: Category;
  label: string;
  variant?: 'dot' | 'solid';
  className?: string;
}

const dotColor: Record<Category, string> = {
  backlog: 'bg-fg-3', unstarted: 'bg-info', started: 'bg-warning',
  completed: 'bg-success', cancelled: 'bg-fg-3',
};
const textColor: Record<Category, string> = {
  backlog: 'text-fg-3', unstarted: 'text-info', started: 'text-warning',
  completed: 'text-success', cancelled: 'text-fg-3 line-through',
};
const solidBg: Record<Category, string> = {
  backlog: 'bg-card', unstarted: 'bg-bg-info', started: 'bg-bg-warning',
  completed: 'bg-bg-success', cancelled: 'bg-card',
};

export function Pill({ category, label, variant = 'solid', className }: PillProps) {
  if (variant === 'dot') {
    return (
      <span className={cn('inline-flex items-center gap-1.5 text-xs', textColor[category], className)}>
        <span className={cn('h-[7px] w-[7px] rounded-full', dotColor[category])} />
        {label}
      </span>
    );
  }
  return (
    <span className={cn('inline-flex items-center gap-1 rounded-pill px-2 py-0.5 text-xs', solidBg[category], textColor[category], className)}>
      {label}
    </span>
  );
}
```

Tests (4): solid renders pale background, dot renders dot+text, both honor `category` color, accepts custom `className`.

### 5.6 `<Tabs>` (new)

`apps/web/src/components/ui/tabs.tsx`. Generic segmented control used by `/login` for Password / Magic / Sign up. Mirrors `mode-toggle.tsx`'s visual.

```tsx
interface TabsProps<T extends string> {
  value: T;
  onChange: (next: T) => void;
  items: Array<{ value: T; label: string }>;
}

export function Tabs<T extends string>({ value, onChange, items }: TabsProps<T>) {
  return (
    <div className="inline-flex items-center gap-0.5 rounded-md border border-border-light bg-shell p-0.5 text-xs">
      {items.map((item) => (
        <button
          key={item.value}
          type="button"
          aria-pressed={value === item.value}
          onClick={() => { if (value !== item.value) onChange(item.value); }}
          className={cn(
            'rounded-sm px-2 py-1',
            value === item.value ? 'bg-primary text-primary-fg' : 'text-fg-2 hover:bg-card',
          )}
        >
          {item.label}
        </button>
      ))}
    </div>
  );
}
```

Tests (3): renders all items, click switches `value`, active item gets primary styling.

### 5.7 `command-palette-bus` (new)

`apps/web/src/lib/command-palette-bus.ts`. Lets non-React or rail code request the palette to open without prop-drilling.

```ts
const EVENT_NAME = 'folio:open-command-palette';

export function openCommandPalette(): void {
  window.dispatchEvent(new CustomEvent(EVENT_NAME));
}

export function subscribeOpenEvent(handler: () => void): () => void {
  window.addEventListener(EVENT_NAME, handler);
  return () => window.removeEventListener(EVENT_NAME, handler);
}
```

`CommandPalette` component subscribes via `useEffect` alongside its existing `keydown` listener.

No tests for this module вҖ” covered indirectly by the existing `command-palette.test.tsx`. (Add one test in wave 2 task 11 that fires the bus event and asserts the palette opens.)
