## Critical lesson from this session — read before debugging anything

**Migrations first when routes look broken.** A long debugging detour today turned out to be: the dev DB was missing migrations 0007–0011. Symptoms looked like route-tree corruption (404s with `text/plain` body), then like server-state bugs (500s on POST/DELETE). Real cause: every `emitEvent` call against a pre-0009 DB threw `table events has no column named seq`. The test suite passed because the test harness creates a fresh `:memory:` DB and migrates from scratch — only the dev DB can be stale.

Fix used: `bun --filter=@folio/server db:migrate`.

Memory file: `~/.claude/projects/-home-ntdst-Projects-folio/memory/feedback_migrations-first-when-routes-look-broken.md`.

**Durable fix not yet applied:** auto-migrate on boot in `apps/server/src/index.ts`. STATE.md references commit `4bf5ff4` as having done this on a prior branch but it's NOT on current main. Without auto-migrate, this bug recurs every time someone pulls a branch with new migrations. Worth a 10-line PR before starting Phase 3.
