### O3 — Spec §3c PATCH approval target_agent recompute — deferred

### O4 — SSE consumer deferred
C8's CommentsTab does NOT subscribe to SSE for live updates from other users/agents. Mutations from the local user update via cache invalidation; remote mutations wait for the 30s staleTime expiry or manual refresh. See `memory/project_realtime-and-locking-deferred.md` and the user decision logged in this session's brief.

**Carry to:** post-2.6 follow-up ticket. Not blocking for D/E close.

### O5 — `data-testid="slideover-activity"` is now a misnomer
After C9 rewrapped slideovers with TabStrip, this test ID lives on the generic tab-content wrapper (Fields/Comments/Activity content area), not the Activity section specifically. A code comment flags this. Future rename pass needed; not blocking.

### O6 — Double agent-document lookup on MCP write tools (from B1 code review)
On agent-bound MCP create/update/delete, the agent row is fetched twice: once in `resolveProjectInWorkspace` (allow-list check) and once in `resolveAuthorContextForToken` (slug lookup). Acceptable for now but worth a cleanup pass. Not blocking.

### O7 — Migrations journal bug pattern
A5 caught a critical bug: A1 shipped migration 0007 without updating `apps/server/src/db/migrations/meta/_journal.json`. Drizzle's `migrate()` silently skipped 0007 in tests AND would have skipped it in production. Tests passed because A1's migration-test harness bypasses the journal and applies SQL directly. See `memory/feedback_drizzle-migration-journal.md`.

**For sub-phase D:** if D adds new migrations, check the journal entry immediately.

### O8 — `copyError` state on CommentRow never clears

### O9 — Soft-delete user label shows raw user ID
The soft-deleted row format for user-authored comments shows `@u-1` (raw ID) instead of resolving to the member's display name. Spec is underspecified; live rows show the resolved name. Flagged in C6 review as Minor. Carry to shake-out.

A small UI call from C9 — the Fields tab for a page document renders an empty-state note. Spec said "skip rendering anything (empty content area)" was acceptable. Cheap to remove if you'd rather it be silent. Decide during smoke.
