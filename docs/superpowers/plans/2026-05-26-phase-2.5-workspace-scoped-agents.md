# Phase 2.5 — Workspace-Scoped Agents Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (sequential — schema → middleware → routes → UI has hard dependencies). Steps use checkbox (`- [ ]`) syntax for tracking. Every task ends with a `testing-workflow:task-complete` gate; every phase ends with a `testing-workflow:phase-complete` gate.

**Goal:** Move agent + trigger identity from project to workspace level with an explicit `projects:` allow-list, removing the project rail leaves and surfacing both from the workspace popover.

**Architecture:** Schema migration relaxes `documents.project_id` to NULL-allowed for `agent`/`trigger` and adds a NOT-NULL `documents.workspace_id`. A new `requireResource` middleware enforces per-request allow-list intersection between the agent's `frontmatter.projects` and the token's optional `project_ids` narrowing. New workspace-level document routes serve agent/trigger CRUD; project-level routes reject those types. MCP resolver gains the same enforcement; `list_projects` filters by allow-list. UI subtracts the project rail leaves and adds Agents/Triggers entries to the workspace popover, with two new pages reusing `DocumentTypeList`.

**Tech Stack:** Bun, Hono, Drizzle (SQLite), Zod, React 18, TanStack Router, TanStack Query, Tailwind, Vitest, Playwright.

**Spec source:** `docs/superpowers/specs/2026-05-26-phase-2.5-workspace-scoped-agents-design.md`

### Step 1.2 — Write the migration SQL

- [ ] Create `apps/server/src/db/migrations/0006_phase_2_5_workspace_agents.sql` with the CREATE+COPY+DROP+RENAME idiom (SQLite cannot ALTER COLUMN to nullable, and cannot add CHECK to existing table). The two table rebuilds run in the order documents → api_tokens because `api_tokens.agent_id` references `documents.id`:

```sql
-- Phase 2.5 — workspace-scoped agents
-- 1. Backfill workspace_id from project, then assert no orphans.
-- 2. Rebuild documents with workspace_id NOT NULL, project_id nullable, CHECK constraint.
-- 3. Rebuild api_tokens with agent_id + project_ids columns.

PRAGMA foreign_keys = OFF;

BEGIN;

-- Step A: add workspace_id nullable to documents and backfill from project.
ALTER TABLE documents ADD COLUMN workspace_id TEXT;
UPDATE documents
   SET workspace_id = (SELECT workspace_id FROM projects WHERE projects.id = documents.project_id);

-- Step B: assert no orphans. If any row still has NULL workspace_id, fail loud.
-- The CREATE TABLE ... NOT NULL on the new table will reject any NULL rows; this is the loud signal.

-- Step C: rebuild documents.
CREATE TABLE documents_new (
  id TEXT PRIMARY KEY NOT NULL,
  project_id TEXT REFERENCES projects(id) ON DELETE CASCADE,
  workspace_id TEXT NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
  table_id TEXT REFERENCES tables(id) ON DELETE SET NULL,
  type TEXT NOT NULL,
  slug TEXT NOT NULL,
  title TEXT NOT NULL,
  status TEXT,
  body TEXT NOT NULL DEFAULT '',
  frontmatter TEXT NOT NULL DEFAULT '{}',
  parent_id TEXT,
  created_by TEXT REFERENCES users(id),
  updated_by TEXT REFERENCES users(id),
  created_at INTEGER NOT NULL DEFAULT (unixepoch() * 1000),
  updated_at INTEGER NOT NULL DEFAULT (unixepoch() * 1000),
  last_touched_at INTEGER,
  CHECK (
    (type IN ('agent','trigger') AND project_id IS NULL)
    OR
    (type IN ('work_item','page') AND project_id IS NOT NULL)
  )
);

-- Pre-rebuild: drop any seeded agent/trigger rows that still have a project_id
-- (Phase 2 seeded them at project level — they violate the new CHECK).
DELETE FROM documents WHERE type IN ('agent','trigger');

INSERT INTO documents_new (
  id, project_id, workspace_id, table_id, type, slug, title, status, body,
  frontmatter, parent_id, created_by, updated_by, created_at, updated_at, last_touched_at
)
SELECT
  id, project_id, workspace_id, table_id, type, slug, title, status, body,
  frontmatter, parent_id, created_by, updated_by, created_at, updated_at, last_touched_at
FROM documents;

DROP TABLE documents;
ALTER TABLE documents_new RENAME TO documents;

CREATE UNIQUE INDEX documents_project_slug_idx ON documents (project_id, slug);
CREATE INDEX documents_project_type_idx ON documents (project_id, type);
CREATE UNIQUE INDEX documents_workspace_type_slug_idx ON documents (workspace_id, type, slug);
CREATE INDEX documents_workspace_type_idx ON documents (workspace_id, type);
CREATE INDEX documents_parent_idx ON documents (parent_id);
CREATE INDEX documents_table_idx ON documents (table_id);

-- Step D: rebuild api_tokens.
CREATE TABLE api_tokens_new (
  id TEXT PRIMARY KEY NOT NULL,
  workspace_id TEXT NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  token_hash TEXT NOT NULL,
  scopes TEXT NOT NULL DEFAULT '[]',
  agent_id TEXT REFERENCES documents(id) ON DELETE CASCADE,
  project_ids TEXT,
  created_by TEXT REFERENCES users(id),
  last_used_at INTEGER,
  created_at INTEGER NOT NULL DEFAULT (unixepoch() * 1000)
);

-- Pre-rebuild: drop tokens whose owning agent we just deleted above.
-- (Phase 2 agent tokens have name like 'agent:<slug>'.)
DELETE FROM api_tokens WHERE name LIKE 'agent:%';

INSERT INTO api_tokens_new (
  id, workspace_id, name, token_hash, scopes, agent_id, project_ids, created_by, last_used_at, created_at
)
SELECT
  id, workspace_id, name, token_hash, scopes, NULL, NULL, created_by, last_used_at, created_at
FROM api_tokens;

DROP TABLE api_tokens;
ALTER TABLE api_tokens_new RENAME TO api_tokens;

CREATE UNIQUE INDEX api_tokens_hash_idx ON api_tokens (token_hash);
CREATE INDEX api_tokens_workspace_idx ON api_tokens (workspace_id);

COMMIT;

PRAGMA foreign_keys = ON;
```

### Step 3.3 — Implement `intersect` + `requireResource` in bearer.ts

- [ ] Append to `apps/server/src/middleware/bearer.ts` (after the existing `requireScope` export at line 70):

```ts
/**
 * Intersect an agent's allow-list with an optional token-level narrowing.
 * - `null` on the token side means "inherit from agent" — distinct from `[]` (no projects).
 * - `'*'` on the agent side means "all workspace projects"; intersection with a concrete list
 *   yields that concrete list.
 */
export function intersect(agentList: string[], tokenList: string[] | null): string[] {
  if (tokenList === null) return agentList;
  if (agentList.includes('*')) return tokenList;
  return agentList.filter((id) => tokenList.includes(id));
}

/**
 * Resource-scope check for bearer requests. Composes after `requireScope`.
 * Session-authenticated requests bypass (workspace membership is the gate).
 * No-op if the route is not project-scoped (no `:pslug`).
 */
export function requireResource(): MiddlewareHandler<AuthContext & { workspace?: unknown; project?: unknown }> {
  return async (c, next) => {
    const token = c.get('token') as { id: string; agentId?: string | null; projectIds?: string[] | null } | undefined;
    const project = c.get('project') as { id: string; slug: string } | undefined;
    if (!token) return next();             // session-authenticated; bypass.
    if (!project) return next();           // no resource to check.
    if (!token.agentId) return next();     // human PAT — Phase 3+ adds enforcement.

    const { db } = await import('../db/client.ts');
    const { documents } = await import('../db/schema.ts');
    const { eq } = await import('drizzle-orm');
    const agent = await db.query.documents.findFirst({ where: eq(documents.id, token.agentId) });
    if (!agent || agent.type !== 'agent') {
      throw new HTTPError('FORBIDDEN_RESOURCE', 'agent for this token no longer exists', 403);
    }
    const agentProjects = ((agent.frontmatter as { projects?: string[] }).projects) ?? ['*'];
    const effective = intersect(agentProjects, token.projectIds ?? null);
    if (!effective.includes('*') && !effective.includes(project.id)) {
      throw new HTTPError(
        'FORBIDDEN_RESOURCE',
        `agent not allow-listed for project ${project.slug}`,
        403,
      );
    }
    return next();
  };
}
```

  ws: Workspace,

  token: { agentId?: string | null; projectIds?: string[] | null },

  const slug = requireString(args, 'project_slug');
  const p = await db.query.projects.findFirst({
    where: and(eq(projects.workspaceId, ws.id), eq(projects.slug, slug)),
  });
  if (!p) throw new Error('project not found');

  if (token.agentId) {
    const agent = await db.query.documents.findFirst({
      where: and(eq(documents.id, token.agentId), eq(documents.type, 'agent')),
    });
    if (!agent) throw mcpInvalidParams('agent for this token no longer exists', { reason: 'agent_missing' });
    const agentProjects = ((agent.frontmatter as { projects?: string[] }).projects) ?? ['*'];
    const effective = intersect(agentProjects, token.projectIds ?? null);
    if (!effective.includes('*') && !effective.includes(p.id)) {
      // Structured server log
      console.info('[mcp] allow-list rejection', {

        agent_id: agent.id,

        requested_project_id: p.id,
        allowed_projects: agentProjects,

      throw mcpInvalidParams(

projectItemRoute.delete('/', async (c) => {
  if (getRole(c) !== 'owner') throw new HTTPError('FORBIDDEN', 'owner only', 403);
  const p = getProject(c);
  const ws = c.get('workspace');

  await db.transaction(async (tx) => {
    // 1. Find every workspace agent/trigger whose frontmatter.projects array contains the deleted project id.
    //    SQLite-side filter: json_array_length + json_extract per element is hairy; do a coarse scan in JS instead.
    const candidates = await tx.query.documents.findMany({

        eq(documents.workspaceId, ws.id),
        // type IN ('agent','trigger')
        // drizzle: use inArray
      ),
    });
    const toRewrite = candidates.filter((d) => {
      if (d.type !== 'agent' && d.type !== 'trigger') return false;
      const projs = (d.frontmatter as { projects?: unknown }).projects;
      return Array.isArray(projs) && projs.includes(p.id);
    });

    for (const doc of toRewrite) {
      const fm = doc.frontmatter as Record<string, unknown>;
      const projs = (fm.projects as string[]).filter((id) => id !== p.id);
      const nextFm = { ...fm, projects: projs };
      await tx.update(documents).set({ frontmatter: nextFm, updatedAt: new Date() }).where(eq(documents.id, doc.id));
    }

    // 2. Delete the project (cascades to its work_items / pages / tables / views / statuses via FK).
    await tx.delete(projects).where(eq(projects.id, p.id));
  });

  return c.body(null, 204);

import { useQuery } from '@tanstack/react-query';
import { client } from './client.ts';
import type { Document, DocumentType } from './documents.ts';

export interface WorkspaceDocsParams {
  type: 'agent' | 'trigger';
  project?: string;
}

export const workspaceDocsKeys = {
  list: (wslug: string, p: WorkspaceDocsParams) => ['workspace-documents', wslug, p] as const,
};

export function useWorkspaceDocuments(
  wslug: string,
  params: WorkspaceDocsParams,
  options: { enabled?: boolean; keepPreviousData?: boolean } = {},
) {
  const search = new URLSearchParams({ type: params.type, ...(params.project ? { project: params.project } : {}) });
  return useQuery({
    queryKey: workspaceDocsKeys.list(wslug, params),
    queryFn: () => client.get<{ items: Document[] }>(`/api/v1/w/${wslug}/documents?${search}`),
    staleTime: 30_000,
    enabled: !!wslug && (options.enabled ?? true),
    placeholderData: options.keepPreviousData ? (prev) => prev : undefined,
  });
}

export function useWorkspaceAgents(wslug: string, opts: { project?: string; enabled?: boolean } = {}) {
  return useWorkspaceDocuments(wslug, { type: 'agent', project: opts.project }, { keepPreviousData: true, enabled: opts.enabled });
}

export function useWorkspaceTriggers(wslug: string, opts: { enabled?: boolean } = {}) {

### Step 9.3 — Implement `ProjectsField`

- [ ] Create `apps/web/src/components/inline/projects-field.tsx`:

```tsx
import { useState } from 'react';
import { Popover, PopoverContent, PopoverTrigger } from '../ui/popover.tsx';

interface Project { id: string; slug: string; name: string; }
interface Props {
  value: string[];
  projects: Project[];
  onChange: (next: string[]) => void;
}

export function ProjectsField({ value, projects, onChange }: Props) {
  const [open, setOpen] = useState(false);
  const isWildcard = value.includes('*');
  const explicit = isWildcard ? projects.map((p) => p.id) : value;
  const selected = new Set(explicit);

  function toggleSelectAll(checked: boolean) {
    onChange(checked ? ['*'] : []);
  }

  function toggleProject(id: string, checked: boolean) {
    if (isWildcard && !checked) {
      // Collapse wildcard to explicit list minus this id.
      onChange(projects.filter((p) => p.id !== id).map((p) => p.id));
      return;
    }
    const next = new Set(selected);
    if (checked) next.add(id);
    else next.delete(id);
    onChange(Array.from(next));
  }

  return (
    <Popover open={open} onOpenChange={setOpen}>
      <PopoverTrigger asChild>
        <div className="flex flex-wrap gap-1.5">
          {isWildcard ? (
            <Chip label="All projects" />
          ) : (
            value.map((id) => {
              const proj = projects.find((p) => p.id === id);
              return <Chip key={id} label={proj?.name ?? `${id.slice(0, 6)}·removed`} muted={!proj} />;
            })
          )}
          {value.length === 0 && <Chip label="No projects" muted />}
        </div>
      </PopoverTrigger>
      <PopoverContent>
        <label className="flex items-center gap-2 px-2 py-1.5 text-sm">
          <input
            type="checkbox"
            aria-label="Select all"
            checked={isWildcard}
            onChange={(e) => toggleSelectAll(e.target.checked)}
          />
          <span>Select all</span>
        </label>
        <div className="border-t border-border" />
        {projects.map((p) => (
          <label key={p.id} className="flex items-center gap-2 px-2 py-1.5 text-sm">
            <input
              type="checkbox"
              aria-label={p.name}
              checked={isWildcard || selected.has(p.id)}
              onChange={(e) => toggleProject(p.id, e.target.checked)}
            />
            <span>{p.name}</span>
          </label>
        ))}
      </PopoverContent>
    </Popover>
  );
}

function Chip({ label, muted }: { label: string; muted?: boolean }) {
  return (
    <span className={`rounded-full px-2 py-0.5 text-xs ${muted ? 'bg-bg-2 text-fg-3' : 'bg-accent/10 text-accent'}`}>
      {label}
    </span>
  );
}
```
